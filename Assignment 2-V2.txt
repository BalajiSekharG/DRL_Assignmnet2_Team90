Question:
The LunarLander-v3 environment in Gymnasium consists of:
	State Space: An 8-dimensional continuous state vector representing the lander's position, velocity, orientation, angular velocity, and leg-contact information. 
	Action Space: 
Action	Description
0	Do nothing
1	Fire left orientation engine
2	Fire main engine
3	Fire right orientation engine
In practical robotic systems, control commands may fail because of hardware faults, communication delays, or actuator unreliability. In this assignment, investigate the effect of stochastic actuator failures on reinforcement learning by creating a modified version of the LunarLander-v3 environment and comparing the performance of Deep Q-Network (DQN) and Double Deep Q-Network (DDQN) agents.
Create a custom environment by wrapping the original LunarLander-v3 environment using gym.Wrapper. The wrapper must modify the action execution process and the reward function while keeping all other aspects of the original environment unchanged.
For every call to step(action), the wrapper must perform the following operations in the given order.
Step 1: Receive the Agent's Action
Let adenote the action selected by the agent from the action space defined above. Store this action before applying any modifications. The stored action will later be used to compute the modified reward.

Step 2: Simulate Intermittent Engine Failure
Before passing the action to the original environment, modify the selected action according to the following rules.
	If a = 0, execute the action without modification. 
	If a ∈ {1,2,3}: 
	Generate a random number r from a uniform distribution in the interval [0,1). 
	If r < 0.15, replace the selected action with 0 (Do Nothing). 
	Otherwise, execute the originally selected action. 
Let a_exec denote the action that is finally executed.
The agent must not receive any indication that its selected action was replaced.
Example
Agent Action (a)	Random Number	Executed Action (a_exec)
0	–	0
1	0.08	0
2	0.63	2
3	0.12	0
3	0.87	3

Step 3: Execute the Action
	Execute a_exec in the original LunarLander-v3 environment.
	Receive the following outputs from the base environment:
	observation 
	base reward (R_base) 
	terminated 
	truncated 
	info 
Step 4: Compute the Modified Reward
Replace the reward returned by the original environment with
R=R_"base" -0.3×1(a≠0)+B
where
	R_"base" is the reward returned by the original environment. 
	a is the action originally selected by the agent (before engine-failure simulation). 
	1(a≠0)is an indicator function that equals 1 when the selected action is 1, 2, or 3, and 0 otherwise. 
	B is the landing bonus defined as 
B={■(50,&"if a safe landing occurs" @0,&"otherwise" )┤

Note
The fuel penalty depends on the action selected by the agent, not on the action actually executed.
Therefore,
	if the agent selects a thruster action and the engine fires successfully, subtract 0.3; 
	if the agent selects a thruster action but the engine misfires and is replaced by Do Nothing, subtract 0.3; 
	if the agent selects Do Nothing, no penalty is applied. 
Step 5: Safe Landing Bonus
Award the landing bonus B=50only if all of the following conditions are satisfied simultaneously:
	terminated == True 
	truncated == False 
	The final observation satisfies: 
	observation[6] == 1 (left landing leg in contact with the landing pad) 
	observation[7] == 1 (right landing leg in contact with the landing pad) 
	|observation[2]| < 0.10 (horizontal velocity) 
	|observation[3]| < 0.10 (vertical velocity) 
	|observation[4]| < 0.10 radians (orientation angle) 



The relevant elements of the observation vector are:
Observation Index	Description
observation[2]	Horizontal velocity
observation[3]	Vertical velocity
observation[4]	Orientation angle
observation[6]	Left landing leg contact (0 = No, 1 = Yes)
observation[7]	Right landing leg contact (0 = No, 1 = Yes)
The landing bonus must be added only if all of the above conditions are satisfied simultaneously. If any condition is not satisfied, including crashes, unstable landings, episode truncation, or incomplete leg contact, no landing bonus shall be awarded. If any of the above conditions is not satisfied, including crashes, unstable landings, episode truncation, or incomplete leg contact, set B=0.
Step 6: Return the Environment Output
Return the following values to the agent:
	observation 
	modified reward (R) 
	terminated 
	truncated 
	info 
No additional information regarding engine failures or reward modifications should be included in the returned info dictionary.
Components That Must Remain Unchanged
Except for the modifications described above,
	the observation space must remain unchanged; 
	the action space must remain unchanged; 
	the state-transition dynamics of the original environment must remain unchanged (except for the stochastic action replacement described above); 
	the episode termination and truncation conditions must remain unchanged; and 
	no additional observations or state variables may be introduced.
